# Module 02 – Instance Personalization

## Overview

This module focuses on basic **instance personalization and branding** within the ServiceNow platform. Administrators can customize elements such as the company logo, company name, and browser title to reflect the organization using the instance.

In this lab, the instance was configured to represent a fictional company called **Dev**, and the changes were verified by impersonating a user.

All exercises were performed in a training instance of **ServiceNow** as part of the ServiceNow Administration Fundamentals (SNAF) course.

---

## Learning Objectives

After completing this module, the following concepts were practiced:

* Customizing instance branding
* Updating the company name
* Modifying the browser title
* Changing the instance logo
* Using user impersonation to verify changes from another user's perspective

---

## Lab Activity – Instance Branding Customization

### Update Company Name

The company name displayed in the instance was updated to represent the fictional organization **Dev**.

This change allows the instance to reflect the identity of the organization using the platform.

---

### Update Browser Title

The browser title was modified so that the name **Dev** appears in the browser tab when accessing the ServiceNow instance.

This helps administrators and users quickly identify the instance they are working in.

---

### Update Company Logo

The default platform logo was replaced with a custom logo representing the **Dev** organization.

Updating the logo helps organizations align the ServiceNow interface with their internal branding.

---

### Verify Changes Using Impersonation

To confirm that the changes were visible to users, impersonation was used.

Steps performed in the lab included:

* Impersonating a company user
* Logging into the platform as that user
* Verifying the updated company name
* Confirming the browser title change
* Confirming the updated logo was visible in the interface

Impersonation allows administrators to view the platform exactly as another user would see it.

---

## Outcome

After completing this lab, the instance successfully reflected the **Dev** branding through the updated company name, browser title, and logo. The changes were verified by impersonating a company user, ensuring the configuration appeared correctly for end users.

This module demonstrates how administrators can personalize the platform to align with organizational branding and improve user recognition of the instance.
