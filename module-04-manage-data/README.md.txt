# Module 04: Manage Data

## Overview
This module focuses on managing and controlling data within the ServiceNow platform. It covers essential administrative tasks such as table management, data import, access control configuration, and an introduction to the Configuration Management Database (CMDB). The module emphasizes maintaining data integrity, security, and proper structuring to support business processes.

## Learning Objectives
- Understand how to manage and maintain tables in ServiceNow
- Configure application-level and user-level access controls
- Import external data into the platform using standard tools
- Understand the structure and purpose of CMDB
- Ensure data consistency, security, and usability

## Key Concepts
- **Table Administration**: Creating, modifying, and maintaining tables and their structure
- **Data Integrity**: Ensuring accurate and consistent data across the platform
- **Access Control (ACLs)**: Defining permissions for users and roles to access data
- **Application Access**: Controlling how applications interact with tables and records
- **Data Import**: Using import sets and transform maps to bring external data into ServiceNow
- **Transform Maps**: Mapping imported data to target tables
- **CMDB (Configuration Management Database)**: Central repository for IT assets and configuration items (CIs)
- **Configuration Items (CIs)**: Individual components tracked in CMDB

## Lab Activities Completed
- Performed table administration tasks including creating and modifying tables
- Configured application access and implemented access control rules (ACLs)
- Imported external data using import sets
- Created and configured transform maps for data mapping
- Loaded data into target tables and validated results
- Explored CMDB structure and worked with configuration items
- Verified relationships and data consistency within CMDB

## Outcome
By completing this module, I gained hands-on experience in managing data within ServiceNow. I am able to administer tables, implement secure access controls, import and transform external data, and understand the foundational concepts of CMDB for managing configuration items effectively.
