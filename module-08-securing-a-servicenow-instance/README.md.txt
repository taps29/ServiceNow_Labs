# Module 08: Securing a ServiceNow Instance

## Overview
This module focuses on securing the ServiceNow platform by understanding shared responsibilities and utilizing built-in security tools. It introduces the Shared Responsibility Model, which defines the roles of ServiceNow and the customer in maintaining security, and explores the ServiceNow Security Center for monitoring and improving instance security posture.

## Learning Objectives
- Understand the Shared Responsibility Model in ServiceNow
- Identify security responsibilities of the platform vs the administrator
- Explore and use the ServiceNow Security Center
- Monitor instance security health and identify vulnerabilities
- Apply best practices to secure the platform

## Key Concepts
- **Shared Responsibility Model**: تقسیم of security responsibilities between ServiceNow (platform security) and customers (configuration, access, and data security)
- **Instance Security**: Protection of data, users, and configurations within the platform
- **Security Center**: Centralized dashboard for monitoring security posture and recommendations
- **Vulnerability Management**: Identifying and addressing potential security risks
- **Access Control**: Managing user roles and permissions to prevent unauthorized access
- **Compliance**: Ensuring the system adheres to organizational and regulatory standards

## Lab Activities Completed
- Explored the Shared Responsibility Model and identified key responsibilities
- Navigated the ServiceNow Security Center
- Reviewed security dashboards and recommendations
- Identified potential security issues within the instance
- Understood how to interpret security scores and insights
- Applied basic security best practices within the platform

## Outcome
By completing this module, I gained a foundational understanding of how to secure a ServiceNow instance. I can identify security responsibilities, use Security Center to monitor system health, and apply best practices to maintain a secure and compliant environment.
